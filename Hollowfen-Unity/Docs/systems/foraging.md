# Foraging System
Walk-up mushroom interaction: HUD prompt → knowledge-gated identification → cinematic focus cam → Inspect journal screen (rotating 3D preview) → two-stick cutting challenge → slot-persistent wild-node cooldown. Harvest access, respawn, buyer values, cultivation, and restoration modifiers are data-authored or derived from canonical project state.
Key scripts: `Assets/_Hollowfen/Scripts/Foraging/` — MushroomNode, MushroomKnowledge, MushroomFieldNotes, MushroomRules, ForageNodeStates, ForageCuttingChallenge/HUD, PlayerInteractor, InspectScreen, InventoryScreen, MushroomPreviewer, InventoryRuntime, MushroomDiscovery, MushroomFocusCamera, IInteractable.
Data: `MushroomFieldGuideData` SOs at `Assets/_Hollowfen/Data/Mushrooms/` (21 species, registered in `Resources/MushroomFieldGuideDatabase.asset` — ALL tied to real fungi with real Latin, educational-game rule). Batch-68 wires 20 entries to species-accurate world + journal prefabs from 16 unique delivered models; same-fungus folk-name entries reuse geometry but retain species-specific `MushroomNode` data. #21 Aldermark = **Hen of the Woods / Maitake (Grifola frondosa)** and still awaits its canon model/photo.
Entry points: `Player/Interact` (E / Triangle) opens Inspect; `Prepare to Cut` starts the brace-and-saw challenge; `Player/OpenInventory` (I / Square) opens Provisions; `P` / L3 opens Wren's Purse and captures a focused Marra/Theo for pouch trade.
Biggest gotchas: every authored wild node needs a stable unique `_nodeId`; story/journal discovery must never reveal an unverified field identity; disable Wren's PlayerInput while screens are open (buffered-Jump bug); ship generated world/journal derivatives, never source-resolution Meshy FBXs.
Status: cinematic harvest, persistent day-based ecology, tactile non-spoiling journal comparison, assembly-reload-safe page navigation, permanent first-find annotations, per-species field verification, four progression tiers, and species-aware trade verified through 2026-07-18.

> Self-healing doc: if you change this system, update this doc (including the 7-line header) in the same batch, and note the change in the batch worksheet.

## Components

| Component | Path | Role |
|---|---|---|
| `IInteractable` | `_Hollowfen/Scripts/Foraging/IInteractable.cs` | Tiny contract — `PromptVerb`, `PromptTarget`, `CanInteract`, `Interact`. Future-proofs NPCs / chests. |
| `MushroomNode` | `_Hollowfen/Scripts/Foraging/MushroomNode.cs` | On the world prefab. References species data plus a stable wild `_nodeId`. `BeginHarvest()` enforces `MushroomRules`, then delegates to the cutting challenge. A successful wild cut records its game day and hides renderers/colliders while keeping the lifecycle host active; the authored respawn dawn restores it. Cultivated nodes opt out of this store. |
| `MushroomRules` | `_Hollowfen/Scripts/Foraging/MushroomRules.cs` | Single policy boundary used by nodes, Inspect, cultivation, and trade. Harvesting requires the persistent per-species `mushroom_identified_<id>` proof plus any authored story-tier flag; journal/story discovery alone is never permission to cut. |
| `ForageNodeStates` | `_Hollowfen/Scripts/Foraging/ForageNodeStates.cs` | Active-slot dictionary of wild node id → harvested day. Availability derives from current day + species cooldown. Persists immediately and refreshes nodes on store or day changes. |
| `ForageCuttingChallenge` | `_Hollowfen/Scripts/Foraging/ForageCuttingChallenge.cs` | Runtime-created controller-first harvest sequence. Plays Wren's retargeted `Forage Kneel` state, runs an unscaled-time side-profile shot, match-cuts to the supplied knife model at the mushroom base, and requires six alternating level strokes (five after Joren's Forge reaches Occupied) while the left stick holds a downward brace. Bad angle or missing brace stalls without punishing the specimen. Standard gamepad motors provide brace resistance, saw texture, stroke ticks, and a final cut pulse. `S` + alternating `A`/`D` provides keyboard parity. Inventory/save state commits only on success; Circle/Esc cancels during the interactive phase. |
| `ForageCuttingHUD` | `_Hollowfen/Scripts/Foraging/ForageCuttingHUD.cs` | Screen-space cinematic lower third (sorting 78): letterbox, short match-cut curtain, live left/right stick wells, brace/angle status, six-notch progress rail, device-specific instructions, and collection confirmation. It intentionally leaves the mushroom and knife as the visual subject instead of using a full-screen modal. |
| `PlayerInteractor` | `_Hollowfen/Scripts/Foraging/PlayerInteractor.cs` | On `PlayerArmature`. Each frame `Physics.OverlapSphereNonAlloc` on the Foraging layer from a chest-height offset, picks the closest `IInteractable` inside a forward cone (dot ≥ -0.2), raises `OnFocusChanged`. Subscribes to `Player/Interact` and calls `current.Interact(gameObject)`. Static `Suspended` flag — any screen that takes input ownership flips it true and search/interact go quiet (HUD fades out automatically). |
| `MushroomDiscovery` | `_Hollowfen/Scripts/Foraging/MushroomDiscovery.cs` | Static active-slot registry of discovered species IDs. Legacy `forage.discoveredIds` PlayerPrefs data migrates once into the slot and is deleted. Fires `OnDiscovered`. |
| `MushroomPreviewer` | `_Hollowfen/Scripts/Foraging/MushroomPreviewer.cs` | Singleton scene component at `(0, -1000, 0)`. Builds its rig in `Awake()` — orthographic camera + key/fill directional lights + turntable mount. Renders to a 1024² ARGB32 RT (4× MSAA). RT background is warm `Parchment` cream so the model reads as a specimen on journal paper. `Show(data, silhouette)` instantiates the SO's `WorldPrefab` onto the mount, recursively re-layers it to `MushroomPreview`, and (when `silhouette=true`) replaces every renderer's `sharedMaterials` with a flat URP-Unlit dark walnut material — used for undiscovered mushrooms so first-encounter inspections show shape + scale only. `ApplyRotationDelta` (yaw + clamped pitch ±80°) and `ApplyZoomDelta` (clamped ortho size) drive manual control; user input disables `AutoRotate` until the next `Show()`. Zoom range `_minOrthoSize = 0.005 .. _maxOrthoSize = 0.50` gives ~26× zoom-in from default 0.13. `SetBackgroundColor(c)` re-applies the camera bg at runtime. |
| `InspectScreen` | `_Hollowfen/Scripts/Foraging/InspectScreen.cs` | Scene-local screen (Map-style, NOT a `UIScreen`). Singleton built programmatically from `UICanvasUtil` + `HollowfenPalette`. Parchment journal layout — clean rounded 1500×940 cream panel, no detached full-rectangle vignette, outer + inner gold-faint double-rule frame. Before field verification, both the node prompt and main sheet remain unnamed and the 3D render stays a silhouette. The first test opens a deterministic three-spread candidate book on a non-target page and keeps the live silhouette beside it. Candidate pages physically fold through a shadowed midpoint with paper rustle and light haptics; the silhouette opens into an optional large observation lens. A wrong match quotes one concise feature from the selected page without naming the target. Passing stages ink observation circles, Wren's cursive note, silhouette-to-full-color model reveal, rarity seal, collection count, stamp audio, and a final haptic/checkmark beat before enabling the cut. The matched spread is held for feature/safety confirmation; later inspections show the real model/details. Quiz answers rest dark and use a bright gold selected state; all mouse/keyboard/gamepad focus paths are explicit. Input/time ownership uses `NarrativePresentationSession`. |
| `MushroomFieldNotes` | `_Hollowfen/Scripts/Foraging/MushroomFieldNotes.cs` | Records the first successful live verification as stable species/day/region/location IDs in a per-slot `GameScores` flag. Display resolves the nearest authored location (within 120m), falls back to the active region, and gives legacy verified saves an undated annotation. The success reveal and Field Guide illustrated spread read the same record. |
| `InteractionPromptHUD` | `_Hollowfen/Scripts/UI/InteractionPromptHUD.cs` | Bottom-center pill under `_HUDCanvas`. Builds itself programmatically. CanvasGroup-driven 120ms fade on unscaled time. Standalone calls publish only after a verified higher-revision inventory save; VillageRequests calls stage under its full-slot transaction, with the wildcard event deferred until commit and counts restored silently on failure. `QuoteFor(MushroomBuyer)` reads accepted/refused/copper totals without mutation; `SellTo` recomputes the same policy, removes only accepted stock, retains refused stock, and persists once. | `InventoryScreen` | `_Hollowfen/Scripts/Foraging/InventoryScreen.cs` | Scene-local screen at `sortingOrder = 55` (below Inspect's 60). Same parchment journal panel as InspectScreen. ## Gameplay profiles and wild lifecycle

Every one of the 21 `MushroomFieldGuideData` assets owns: `ForageTier`, optional `RequiredForageFlagId`, `WildRespawnDays`, per-item Marra/Theo values, and cultivation fields. `MushroomGameplayFoundationImporter` is the canonical balance table and stamps profiles, dialogue buyer routing, scene node IDs, and the mill rest point idempotently.

The scene currently contains 21 authored wild nodes across the species already placed during world dressing. All node IDs are unique (`wild.<species>.<nnn>`). The other profiled species are engine-ready but still belong to the planned world-dressing placement pass.

## Gotchas / conventions

- **Localization keys, not raw strings, on `IInteractable.PromptVerb`.** `MushroomNode.PromptVerb => "prompt.inspect.verb"`; the HUD calls `Localization.Get` on it. Keeps the HUD a single funnel for new verbs (NPC "Talk", chest "Open", etc.).
- **Two layers added**: `Foraging` (slot 8) hosts only the trigger SphereCollider on the prefab root — the inner `Mesh` child with the convex MeshCollider stays on `Default` so movement physics doesn't collide with the focus-search lane. `MushroomPreview` (slot 9) isolates the previewer's camera + spawned model from the main camera.
- **PlayerArmature was already on layer 8** from the StarterAssets template. OverlapSphere returns the player's own collider every frame but `GetComponentInParent<IInteractable>` filters it out — slightly wasteful, not a bug.
- **InspectScreen auto-creates an EventSystem** if none exists (so playing directly into `Scene_Hollowfen` works) and turns off `InputSystemUIInputModule.deselectOnBackgroundClick`. `Update()` re-applies the last-focused button if `currentSelectedGameObject` goes null.
- **Identity reveal boundary**: `MushroomDiscovery` and `CanReadPage` expose reference knowledge, not the live specimen's name. `InspectScreen.ShowInternal`, its details sheet, `MushroomPreviewer.Show(..., silhouette)`, and `MushroomNode.PromptTarget` all reveal identity only through `MushroomKnowledge.IsFieldIdentified`. Do not regress one surface back to story discovery.
- **First-test page browser**: candidate spreads are the target plus two deterministic readable distractors; the browser deliberately starts on a distractor when possible. Match is evaluated against the current page, not a species-name answer label. Page turning resets a prior wrong-page treatment, the correct spread locks for the two confirmation stages, and the enlarged gallery exposes Previous/Next only while browsing is still part of the question.
- **Discovery choreography is state-safe**: identity is committed before the ceremony begins, but **Prepare to Cut** remains disabled until the unscaled reveal finishes. Cancellation may close presentation without rolling back a valid identification. Page-turn and reveal coroutines, rumble, overlays, and preview state are stopped from every close/destroy path.
- **In-Play assembly reload recovery**: Unity can preserve the code-built journal hierarchy while clearing its static singleton, generated input wrapper, and runtime-only `Button.onClick` delegates. `InspectScreen.OnEnable` detects that exact state, restores its singleton/input/presentation lease, rebinds persistent controls, and reconstructs the current quiz stage. The page-turn iterator also restores leaf scale and both navigation sets in `finally`, so a reload or interrupted coroutine cannot leave Previous/Next disabled.
- **First-find memory**: `mushroom_field_note|<species>|<day>|<region>|<location>` is an encoded per-slot flag. Do not localize or rename its stored IDs. `MushroomFieldNotes.PlaceName` performs localization at display time; previously verified saves without a note intentionally show a graceful legacy line instead of fabricating a date/place.
- **Forage shortcut redundancy**: inside the inspect screen, `Player/Interact` (Triangle / E) is a Forage shortcut, `UI/Submit` activates the focused button, `UI/Cancel` leaves. Same Triangle button opens the screen and forages — feels continuous.
- **Repeatable model pipeline (batch-68)**: `MushroomModelManifest.json` is the single mapping for 16 unique source drops → 20 canonical species. Run `tools/optimize_mushroom_models.py` through Blender to generate world (12k–16k triangles) and journal (60k–75k triangles) FBXs, then run `Hollowfen/Assets/Rebuild Mushroom Models`. The importer packs `metallic.r + (1 - roughness.r).a` into a linear URP mask map, caps shipping texture imports at 1024, creates shared instanced URP Lit materials, rebuilds prefabs idempotently, and rewires the SOs.
- **Mushroom prefab pivot + scale**: generated prefab roots stay at scale 1. The importer computes renderer bounds, normalizes ground species by authored height (flat/mounted clusters by maximum extent), centers X/Z, grounds Y when appropriate, and adds a `Foraging`-layer trigger plus a Default-layer box collision child. This replaces the fragile per-export `localScale=0.08` recipe while preserving existing prefab GUIDs and scene placements.
- **`MushroomPreviewer.Update()` uses `Time.unscaledDeltaTime`** so auto-rotate keeps spinning while the inspect screen is paused.
- **Silhouette material is created lazily** on first `Show(data, silhouette=true)` — `Shader.Find("Universal Render Pipeline/Unlit")` with `_BaseColor` (+ URP-fallback `_Color`) near-black walnut. Reverting is automatic — the discovered path re-instantiates the prefab with original materials.
- **Inspect-screen layout dims**: panel 1500×940, preview 700×700 anchored `(56, 60)` panel-local (upper half — clears the centered button row at `(0, 36)` from panel bottom), text column starts at `868` from panel left. Stat strip is 170 tall because `Mushroom_07_FieldMushroom` LOOK-ALIKES runs ~7 lines. Hint pill at TOP of preview frame (bottom collided with buttons at high widths).
- **Gamepad glyph detection** uses `Gamepad.current.GetType().Name` + `pad.description.product.ToLowerInvariant()` matched against PS / Xbox / Switch / generic. Default fallback is PS-style (`△`/`○`). Glyphs refresh every frame.
- **Foraging gating**: every encounter routes through Inspect — no quick-pickup. Every one of the 21 species, including story-taught Field Mushroom, Wood Ear, and Pinecrest, must pass its own three-stage identification once before harvest. `MushroomDiscovery` exposes journal/name knowledge; `MushroomKnowledge.IsFieldIdentified` reads the separately persisted proof. A locked tier remains inspectable but its action is disabled with an authored knowledge note. Both the UI and node entry paths recheck `MushroomRules.CanHarvest`; harvest itself never creates discovery/verification state.
- **First-harvest achievement** (`ACH_FORAGE_FIRST`) is PlayerPrefs-gated by `forage.firstHarvestSeen`.
- **Mesh budget evidence (batch-68)**: the source Field Mushroom was 414k vertices; its shipping world derivative is 15,803 vertices and journal derivative 47,378 vertices. Source art remains untouched outside the Unity runtime path.
- **Inventory persistence**: `MushroomNode.Harvest` → `InventoryRuntime.Add(_data, 1)` → snapshot written to the current `SaveManager.ActiveSlot` immediately. Wild state is written separately through `ForageNodeStates`. Dev reset should use `SaveCoordinator.StartNewGame`; deleting only slot 0 is insufficient after loading another slot.
- **Cinematic cutting sequence** (`MushroomNode.BeginHarvest`): `InspectScreen.HideForCinematic` deliberately leaves pause/input ownership to `ForageCuttingChallenge`. The challenge slows the world to `0.12` while Wren's Animator and Cinemachine Brain use unscaled time, plays the 2.767s kneeling clip, then uses a 160ms black match cut to a clean macro shot. Wren's renderer is hidden only for the isolated knife close-up and restored before gameplay resumes. `RestorationBenefits.CuttingStrokes` supplies the five/six target; the release beat commits and lifts the mushroom before deactivation. Cleanup restores exact Animator, renderer, Brain, time-scale, PlayerInput, interactor, focus-camera, and rumble state.

- **Restored wild ecology:** the Witch's Cottage Occupied benefit projects a three-day species cooldown to two days (and never below one). Node persistence remains harvested-day only, so existing saves automatically receive the benefit without rewriting their forage snapshot.
- **Knife scale rule**: the supplied FBX mesh is centimeter-authored (`~0.019` imported mesh units long). Runtime sizing uses mesh-local bounds rather than same-frame `Renderer.bounds`, then normalizes the knife to 34–48cm based on specimen height. This avoids the first-frame stale-bounds bug that rendered it at 1/100 intended size.
- **`PlayerInteractor.SetPlayerInputEnabled(bool)`** toggles Wren's StarterAssets `PlayerInput`. Used by InspectScreen, InventoryScreen, and the harvest cinematic. Without it, Cross/Space confirms Forage AND queues a buffered `Player/Jump` that fires when timeScale resumes — Wren jumps after every harvest.
- **`PlayerInteractor.DismissCurrent()`** backs out of cinematic zoom (`MushroomFocusCamera.OnCancel`). Saves the target into `_suppressed`, clears focus; suppression clears once the target leaves OverlapSphere range.
- **Oyster placement**: mounted to trunks via `Quaternion.FromToRotation(Vector3.up, outwardNormal)` — Meshy's Oyster growth axis is local +Y. `Mushroom_17_Oyster` was authored from scratch (web import had 16 species) as the 17th database entry.
- **Preview centering**: Meshy exports pivot at mesh BASE. `MushroomPreviewer.Show` computes renderer bounds and shifts the child so `bounds.center == mount.position` — otherwise the camera frames the stem base.
- **Camera pan in previewer**: `ApplyPanDelta(Vector2)` shifts camera position AND look target by the same world-space vector along `cam.right/up` — true lateral pan, composes with rotate + zoom. `ResetView()` zeroes pan/yaw/pitch/ortho on every `Show()`.
