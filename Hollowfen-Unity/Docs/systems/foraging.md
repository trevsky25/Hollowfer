# Foraging System
Walk-up mushroom interaction: HUD prompt → cinematic focus cam → Inspect journal screen (rotating 3D preview) → two-stick cutting challenge → slot-persistent wild-node cooldown. Harvest access, respawn, buyer values, and cultivation are data-authored per species.
Key scripts: `Assets/_Hollowfen/Scripts/Foraging/` — MushroomNode, MushroomRules, ForageNodeStates, ForageCuttingChallenge/HUD, PlayerInteractor, InspectScreen, InventoryScreen, MushroomPreviewer, InventoryRuntime, MushroomDiscovery, MushroomFocusCamera, IInteractable.
Data: `MushroomFieldGuideData` SOs at `Assets/_Hollowfen/Data/Mushrooms/` (21 species, registered in `Resources/MushroomFieldGuideDatabase.asset` — ALL tied to real fungi with real Latin, educational-game rule). Batch-68 wires 20 entries to species-accurate world + journal prefabs from 16 unique delivered models; same-fungus folk-name entries reuse geometry but retain species-specific `MushroomNode` data. #21 Aldermark = **Hen of the Woods / Maitake (Grifola frondosa)** and still awaits its canon model/photo.
Entry points: `Player/Interact` (E / Triangle) opens Inspect; `Prepare to Cut` starts the brace-and-saw challenge; `Player/OpenInventory` (I / Square) opens Provisions; `P` / L3 opens Wren's Purse and captures a focused Marra/Theo for pouch trade.
Biggest gotchas: every authored wild node needs a stable unique `_nodeId`; disable Wren's PlayerInput while screens are open (buffered-Jump bug); ship generated world/journal derivatives, never source-resolution Meshy FBXs.
Status: cinematic harvest plus persistent day-based ecology, four progression tiers, and species-aware trade play-mode verified 2026-07-16.

> Self-healing doc: if you change this system, update this doc (including the 7-line header) in the same batch, and note the change in the batch worksheet.

## Gameplay profiles and wild lifecycle

Every one of the 21 `MushroomFieldGuideData` assets owns: `ForageTier`, optional `RequiredForageFlagId`, `WildRespawnDays`, per-item Marra/Theo values, and cultivation fields. `MushroomGameplayFoundationImporter` is the canonical balance table and stamps profiles, dialogue buyer routing, scene node IDs, and the mill rest point idempotently.

The scene currently contains 21 authored wild nodes across the species already placed during world dressing. All node IDs are unique (`wild.<species>.<nnn>`). The other profiled species are engine-ready but still belong to the planned world-dressing placement pass.

## Gotchas / conventions

- **Localization keys, not raw strings, on `IInteractable.PromptVerb`.** `MushroomNode.PromptVerb => "prompt.inspect.verb"`; the HUD calls `Localization.Get` on it. Keeps the HUD a single funnel for new verbs (NPC "Talk", chest "Open", etc.).
- **Two layers added**: `Foraging` (slot 8) hosts only the trigger SphereCollider on the prefab root — the inner `Mesh` child with the convex MeshCollider stays on `Default` so movement physics doesn't collide with the focus-search lane. `MushroomPreview` (slot 9) isolates the previewer's camera + spawned model from the main camera.
- **PlayerArmature was already on layer 8** from the StarterAssets template. OverlapSphere returns the player's own collider every frame but `GetComponentInParent<IInteractable>` filters it out — slightly wasteful, not a bug.
- **InspectScreen auto-creates an EventSystem** if none exists (so playing directly into `Scene_Hollowfen` works) and turns off `InputSystemUIInputModule.deselectOnBackgroundClick`. `Update()` re-applies the last-focused button if `currentSelectedGameObject` goes null.
- **Forage shortcut redundancy**: inside the inspect screen, `Player/Interact` (Triangle / E) is a Forage shortcut, `UI/Submit` activates the focused button, `UI/Cancel` leaves. Same Triangle button opens the screen and forages — feels continuous.
- **Repeatable model pipeline (batch-68)**: `MushroomModelManifest.json` is the single mapping for 16 unique source drops → 20 canonical species. Run `tools/optimize_mushroom_models.py` through Blender to generate world (12k–16k triangles) and journal (60k–75k triangles) FBXs, then run `Hollowfen/Assets/Rebuild Mushroom Models`. The importer packs `metallic.r + (1 - roughness.r).a` into a linear URP mask map, caps shipping texture imports at 1024, creates shared instanced URP Lit materials, rebuilds prefabs idempotently, and rewires the SOs.
- **Mushroom prefab pivot + scale**: generated prefab roots stay at scale 1. The importer computes renderer bounds, normalizes ground species by authored height (flat/mounted clusters by maximum extent), centers X/Z, grounds Y when appropriate, and adds a `Foraging`-layer trigger plus a Default-layer box collision child. This replaces the fragile per-export `localScale=0.08` recipe while preserving existing prefab GUIDs and scene placements.
- **`MushroomPreviewer.Update()` uses `Time.unscaledDeltaTime`** so auto-rotate keeps spinning while the inspect screen is paused.
- **Silhouette material is created lazily** on first `Show(data, silhouette=true)` — `Shader.Find("Universal Render Pipeline/Unlit")` with `_BaseColor` (+ URP-fallback `_Color`) near-black walnut. Reverting is automatic — the discovered path re-instantiates the prefab with original materials.
- **Inspect-screen layout dims**: panel 1500×940, preview 700×700 anchored `(56, 60)` panel-local (upper half — clears the centered button row at `(0, 36)` from panel bottom), text column starts at `868` from panel left. Stat strip is 170 tall because `Mushroom_07_FieldMushroom` LOOK-ALIKES runs ~7 lines. Hint pill at TOP of preview frame (bottom collided with buttons at high widths).
- **Gamepad glyph detection** uses `Gamepad.current.GetType().Name` + `pad.description.product.ToLowerInvariant()` matched against PS / Xbox / Switch / generic. Default fallback is PS-style (`△`/`○`). Glyphs refresh every frame.
- **Foraging gating**: every encounter routes through Inspect — no quick-pickup. A locked tier remains inspectable but its Forage button is disabled with an authored knowledge note. Both the UI and commit path recheck `MushroomRules.CanHarvest`.
- **First-harvest achievement** (`ACH_FORAGE_FIRST`) is PlayerPrefs-gated by `forage.firstHarvestSeen`.
- **Mesh budget evidence (batch-68)**: the source Field Mushroom was 414k vertices; its shipping world derivative is 15,803 vertices and journal derivative 47,378 vertices. Source art remains untouched outside the Unity runtime path.
- **Inventory persistence**: `MushroomNode.Harvest` → `InventoryRuntime.Add(_data, 1)` → snapshot written to the current `SaveManager.ActiveSlot` immediately. Wild state is written separately through `ForageNodeStates`. Dev reset should use `SaveCoordinator.StartNewGame`; deleting only slot 0 is insufficient after loading another slot.
- **Cinematic cutting sequence** (`MushroomNode.BeginHarvest`): `InspectScreen.HideForCinematic` deliberately leaves pause/input ownership to `ForageCuttingChallenge`. The challenge slows the world to `0.12` while Wren's Animator and Cinemachine Brain use unscaled time, plays the 2.767s kneeling clip, then uses a 160ms black match cut to a clean macro shot. Wren's renderer is hidden only for the isolated knife close-up and restored before gameplay resumes. Six clean strokes commit state; the 720ms release beat tilts/lifts/scales the mushroom before deactivation. Cleanup restores exact Animator, renderer, Brain, time-scale, PlayerInput, interactor, focus-camera, and rumble state.
- **Knife scale rule**: the supplied FBX mesh is centimeter-authored (`~0.019` imported mesh units long). Runtime sizing uses mesh-local bounds rather than same-frame `Renderer.bounds`, then normalizes the knife to 34–48cm based on specimen height. This avoids the first-frame stale-bounds bug that rendered it at 1/100 intended size.
- **`PlayerInteractor.SetPlayerInputEnabled(bool)`** toggles Wren's StarterAssets `PlayerInput`. Used by InspectScreen, InventoryScreen, and the harvest cinematic. Without it, Cross/Space confirms Forage AND queues a buffered `Player/Jump` that fires when timeScale resumes — Wren jumps after every harvest.
- **`PlayerInteractor.DismissCurrent()`** backs out of cinematic zoom (`MushroomFocusCamera.OnCancel`). Saves the target into `_suppressed`, clears focus; suppression clears once the target leaves OverlapSphere range.
- **Oyster placement**: mounted to trunks via `Quaternion.FromToRotation(Vector3.up, outwardNormal)` — Meshy's Oyster growth axis is local +Y. `Mushroom_17_Oyster` was authored from scratch (web import had 16 species) as the 17th database entry.
- **Preview centering**: Meshy exports pivot at mesh BASE. `MushroomPreviewer.Show` computes renderer bounds and shifts the child so `bounds.center == mount.position` — otherwise the camera frames the stem base.
- **Camera pan in previewer**: `ApplyPanDelta(Vector2)` shifts camera position AND look target by the same world-space vector along `cam.right/up` — true lateral pan, composes with rotate + zoom. `ResetView()` zeroes pan/yaw/pitch/ortho on every `Show()`.
