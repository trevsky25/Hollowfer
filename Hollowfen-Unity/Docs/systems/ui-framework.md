# UI Framework
Screen-stack UI system: `UIManager` singleton (DDOL) owns a push/pop stack of `UIScreen`s, fade transitions, scene-transition orchestration, and pause input. All UI is built programmatically — screen GameObjects are empty Canvas hosts; `OnInitialize` constructs everything from code.
Key scripts: `Assets/_Hollowfen/Scripts/UI/` — UIManager/UIScreen, menu screens, shared journal family, NarrationOverlay/IntroGuide, StoryMomentDirector, and runtime screens including EndingCredits and RestorationLedger; `Scripts/Restoration/RestorationRevealDirector.cs` owns the restoration lower third; `Scripts/Editor/StoryArtQualityImporter.cs` owns story-painting import fidelity.
Scene flow: `Scene_MainMenu` (boot, DDOL menu screens) ⇄ `Scene_Hollowfen` (gameplay) via `UIManager.LoadSceneAndOpen`.
Entry points: any screen change goes through `UIManager.Push/Pop`; scene changes through `LoadSceneAndOpen(sceneName, nextScreenId)`.
Biggest gotchas: configure runtime focus visuals with `FocusHighlight.Configure(...)`; prefer `RectMask2D`; avoid uGUI Outline on sliced sprites; runtime screens must call `ConfigureRuntimeScreen` before registration. Presentation state is lease-owned—see `presentation-sessions.md`—and shortcut input is blocked while any presentation or screen transition owns it. Overlay cards must not combine rounded fills with detached shadows or unclipped full-rectangle decoration. Root screens must declare `IsRootScreen`; every `ScaleWithScreenSize` canvas must use the accessibility policy's current reference resolution.
Status: production-audited through 2026-07-18. Root/modal shape rules, identification recovery, the seven-project controller ledger, restoration lower thirds, 100/108/115% interface scaling, reduced-motion alternatives, and caption backing are runtime-verified; standard and largest settings/ledger layouts received live visual inspection.

> Self-healing doc: if you change this system, update this doc (including the 7-line header) in the same batch, and note the change in the batch worksheet.

## Scene flow

```
Scene_MainMenu  ──Continue──►  Scene_Hollowfen  ──Pause→Quit──►  Scene_MainMenu
       │                              │
   (boot scene)               (gameplay village; Wren spawns)
       │                              │
       └─ MainMenu / Settings / Save Slot / Pause / ConfirmModal / Loading
```

- `Scene_MainMenu` (`Assets/_Hollowfen/Scenes/Scene_MainMenu.unity`) is the boot scene: UIManager + ScreenRoot + all menu UIScreens as DDOL'd children.
- `Scene_Hollowfen` (`Assets/_Hollowfen/Scenes/Scene_Hollowfen.unity`) is the actual village — a copy of Magic Pig Games' "Medieval Environment - Demo 1" moved into our owned folder so package updates don't clobber it. Contains `PlayerArmature` (Wren) with `ThirdPersonController` + `StarterAssetsInputs` + `PlayerInput` on `Assets/Starter Assets/.../StarterAssets.inputactions`. The original demo stays untouched at its package path.
- Legacy prototype `Assets/Scenes/MainMenu.unity` is visual reference only (intentional design — port the look, don't regenerate art).
- Build Settings order (batch-34 cleanup): Scene_MainMenu (0), Scene_Hollowfen. The legacy `MainMenu`/`Village` prototype scenes were dropped from the build.

## Gotchas

- **All UI built programmatically.** `_built` flag prevents rebuilding on repeat OnInitialize. Re-running the scene rebuilds from current code — no prefabs to keep in sync.
- **Use explicit Unity component creation on runtime screen hosts.** `GetComponent<T>() ?? AddComponent<T>()` can retain a Unity fake-null wrapper across editor/runtime edges; use the established `var c = GetComponent<T>(); if (c == null) c = AddComponent<T>();` pattern before dereferencing Canvas, CanvasScaler, CanvasGroup, or raycaster components.
- **Root canvas contract:** every authored screen/HUD root uses `ScaleWithScreenSize`, `AccessibilityPresentationPolicy.ReferenceResolution`, match `0.5`. That reference is 1920×1080 at 100% and proportionally smaller at 108/115%. If a legacy composition must retain 1280×800 geometry, put it under an explicit presentation root; do not leave a second scaler convention in the scene.
- **A hidden HUD is passive.** Setting `CanvasGroup.alpha=0` is not enough: presentation-only HUD roots keep `interactable=false` and `blocksRaycasts=false` so invisible layers cannot consume input.
- **Fade shadows with their surface.** `UICanvasUtil.AddShadow` creates a sibling of its target. Put both under the same presentation-level CanvasGroup when the surface animates; the cutting HUD's `CuttingPanelPresentation` is the reference implementation.
- **A rounded fill does not clip rectangular child images.** Full-card gradients, grain, and sheen remain rectangular unless explicitly clipped. On overlay cards, prefer one rounded fill plus inset accents; do not add an unclipped full-card decoration or detached `SoftShadow` merely for depth.
- **Animate the journal leaf, not the whole modal.** Identification page turns scale the art leaf on X and sweep a contained shadow inside it; controls are disabled during the 0.36-second fold/unfold. This preserves the cream rounded shell and prevents a square animation layer from reintroducing the popup artifact.
- **Never pop an application root.** Root screens override `IsRootScreen`; `UIManager.Back()` ignores direct root-pop requests, while the root's own `OnBack()` may present an explicit exit confirmation.
- **`FocusHighlight._baseColor` caches at Awake.** Call `Configure(...)` immediately after `AddComponent`; it assigns the target and base/focus colors atomically. Reflection mutation is legacy-only and should not appear in new screens.
- **Reduced Motion still needs a selected-state cue.** Suppress scale travel only; keep the authored focus color, outline, glow, and controller selection. Cinematics must still clamp to their final composition before callbacks.
- **A RenderTexture's ARGB format does not make its background transparent by itself.** The preview camera must also clear with alpha `0`; place any desired stage/halo in the UI beneath the `RawImage`, not in camera clear color.
- **`Outline` is a `BaseMeshEffect`, not a `Graphic`.** Assigning it to `FocusHighlight._targetGraphic` throws — use a real Graphic (Image) overlay.
- **`Mask` + near-transparent Image clips everything.** Use `RectMask2D` for scroll viewports.
- ~~**Georgia SDF loads via `AssetDatabase` editor-only**~~ **RESOLVED batch-55**: `UIManager.Awake` calls `UICanvasUtil.SetHeadingFont/SetBodyFont` from `[SerializeField]` refs (IMFellEnglish SDF / EBGaramond SDF) on the scene UIManager, so code-built headings resolve in a player build. The editor-only `AssetDatabase` path in `HeadingFont` remains only as a fallback for screens built before UIManager.Awake.
- **CanvasRenderer + reflection AddComponent** has bitten us before (see memory/gotchas); prefer explicit component setup.
- Use `Time.timeScale` for pause; UI animations run on unscaled time.
