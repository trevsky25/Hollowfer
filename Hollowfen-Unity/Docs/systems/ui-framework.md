# UI Framework
Screen-stack UI system: `UIManager` singleton (DDOL) owns a push/pop stack of `UIScreen`s, fade transitions, scene-transition orchestration, and pause input. All UI is built programmatically — screen GameObjects are empty Canvas hosts; `OnInitialize` constructs everything from code.
Key scripts: `Assets/_Hollowfen/Scripts/UI/` — UIManager, UIScreen, MainMenuScreen, SettingsScreen, SaveSlotScreen, ConfirmModal, LoadingScreen, FocusHighlight, UICanvasUtil, HollowfenPalette; scene-local overlays NarrationOverlay (sort 70; two modes — `Show(...)` = captions on full black for the act-break journal narration; `ShowCinematic(...)` = batch-36 opening, captions in the lower third over a Ken-Burned hero image with letterbox bars + a full-screen vignette scrim. **batch-41**: the multi-image overload `ShowCinematic(captions, clips, Sprite[] heroes, int[] beatImage, onDone)` plays an ordered sequence of paintings that **crossfade-dissolve** (two hero layers A/B, deterministic sibling order), each with its own motivated **Ken Burns** from a per-image `DefaultMoves` table; single/two-image overloads still exist and now route through the same dissolve path. The homecoming intro (`StoryBeats._introImages`) uses it with the 4 bespoke `intro-0N-*.png` paintings — ridge→wrong-river→boarded-cottages→silent-square/Bram — over the bible's Scene-1 passage; image-0's KB A-state matches LoadingScreen for the seamless handoff) + IntroGuide (sort 65, batch-30 first-steps journal page, restyled batch-31: offset-LEFT page with the world visible right, `UICanvasUtil.PaperGrain` texture + inset gold frame + ledger double-rule + keycap chips; once per save via `intro_guide_seen`, triggered by StoryBeats after the intro).
Scene flow: `Scene_MainMenu` (boot, DDOL menu screens) ⇄ `Scene_Hollowfen` (gameplay) via `UIManager.LoadSceneAndOpen`.
Entry points: any screen change goes through `UIManager.Push/Pop`; scene changes through `LoadSceneAndOpen(sceneName, nextScreenId)`.
Biggest gotchas: FocusHighlight `_baseColor` caches at Awake (reflection reassignment trap); use `RectMask2D` not `Mask` for scroll viewports; **`Mask` + URP UI renders the mask graphic CYAN** (batch-47 — clip by composition instead, e.g. inset the content within the rounded shape); **uGUI `Outline` component + sliced sprite = gray wash** (it re-draws the 9-slice 4× offset — use a `RoundifyOutline` hairline child instead); HUD components that build onto scene GameObjects must WIPE existing children at build or stale serialized UI renders behind them (batch-47 — the prompt's square box); `UICanvasUtil.Roundify/RoundifyOutline` are the one-line rounding helpers for existing Images; controller button icons = `ControllerGlyphs.For(Face)` (batch-48 — PS pads get `<sprite name="ps_*">` tags from the procedural ControllerGlyphs TMP sprite sheet, regenerate via menu Hollowfen→Generate Controller Glyphs; a TMP_SpriteAsset saved without `m_Version="1.1.0"` gets its tables WIPED by TMP's legacy upgrade on load, and `TMP_Settings.defaultSpriteAsset` must be persisted via SerializedObject); ~~Georgia SDF ship blocker~~ FIXED batch-32, then **Georgia RETIRED batch-55** — migrated to OFL fonts **IM Fell English SDF** (titles) + **EBGaramond SDF** (body/UI, = TMP default); `UIManager.Awake` injects both into `UICanvasUtil` (runtime-safe, fixed the editor-only-heading latent bug); see conventions.md Fonts rule; UIManager's PUSH deactivates the covered screen WITHOUT calling OnClose — a covered screen's InputActions stay enabled, so screen-level handlers must gate on `UIManager.Instance.TopScreen == this` (batch-28 review catch); `OnPauseInput` bails while NarrationOverlay/IntroGuide show (their canvases sort ABOVE the screen stack — a pause under them would be an invisible modal that steals focus and can leak `PlayerInteractor.Suspended` via quit; batch-30 review catch).
Status: shipped + verified. Pause prefab root is a plain Transform — build UI into a Canvas child. SettingsScreen rebuilt code-built in batch-28, ConfirmModal in batch-44 (parchment card, wipes its legacy canvas at OnInitialize — API unchanged; menu chrome still on legacy scene-UI: MainMenu, SaveSlot, Loading — the TMP-migration backlog). Batch-44 also added `UISfx` (procedural synthesized click, no audio assets) — UIManager.TransitionRoutine ticks on every push/replace/pop except boot-open and loading-screen transitions, routed via UIManager._uiSfxOutput → SFX mixer group (respects the SFX slider).

## Scene flow

```
Scene_MainMenu  ──Continue──►  Scene_Hollowfen  ──Pause→Quit──►  Scene_MainMenu
       │                              │
   (boot scene)               (gameplay village; Wren spawns)
       │                              │
       └─ MainMenu / Settings / Save Slot / Pause / ConfirmModal / Loading
```

- `Scene_MainMenu` (`Assets/_Hollowfen/Scenes/Scene_MainMenu.unity`) is the boot scene: UIManager + ScreenRoot + all menu UIScreens as DDOL'd children.
- `Scene_Hollowfen` (`Assets/_Hollowfen/Scenes/Scene_Hollowfen.unity`) is the actual village — a copy of Magic Pig Games' "Medieval Environment - Demo 1" moved into our owned folder so package updates don't clobber it. Contains `PlayerArmature` (Wren) with `ThirdPersonController` + `StarterAssetsInputs` + `PlayerInput` on `Assets/Starter Assets/.../StarterAssets.inputactions`. The original demo stays untouched at its package path.
- Legacy prototype `Assets/Scenes/MainMenu.unity` is visual reference only (intentional design — port the look, don't regenerate art).
- Build Settings order (batch-34 cleanup): Scene_MainMenu (0), Scene_Hollowfen. The legacy `MainMenu`/`Village` prototype scenes were dropped from the build.

## Gotchas

- **All UI built programmatically.** `_built` flag prevents rebuilding on repeat OnInitialize. Re-running the scene rebuilds from current code — no prefabs to keep in sync.
- **`FocusHighlight._baseColor` caches at Awake.** Reassigning `_targetGraphic` via reflection AFTER AddComponent paints the old target's cached color onto the new target on first state change (symptom: dark/light overlay across the control). Fix: also set `_baseColor` via reflection AND snap the target's color to the new resting value immediately.
- **`Outline` is a `BaseMeshEffect`, not a `Graphic`.** Assigning it to `FocusHighlight._targetGraphic` throws — use a real Graphic (Image) overlay.
- **`Mask` + near-transparent Image clips everything.** Use `RectMask2D` for scroll viewports.
- ~~**Georgia SDF loads via `AssetDatabase` editor-only**~~ **RESOLVED batch-55**: `UIManager.Awake` calls `UICanvasUtil.SetHeadingFont/SetBodyFont` from `[SerializeField]` refs (IMFellEnglish SDF / EBGaramond SDF) on the scene UIManager, so code-built headings resolve in a player build. The editor-only `AssetDatabase` path in `HeadingFont` remains only as a fallback for screens built before UIManager.Awake.
- **CanvasRenderer + reflection AddComponent** has bitten us before (see memory/gotchas); prefer explicit component setup.
- Use `Time.timeScale` for pause; UI animations run on unscaled time.
